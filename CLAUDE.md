## 1. PRIME DIRECTIVE

### Maximum Leverage. Minimum Surface Area. Zero Entropy.

Reject **any** architectural expansion unless it produces:

- ≥2× resilience gain
- ≥2× clarity gain
- ≥2× capability gain
- or measurable operational simplification

Every addition must:

- justify its existence with a rationale
- expose telemetry signals (HEALTH / PRESSURE / EFFICIENCY / FAILURE / TRACE)
- reduce future entropy — not defer it
- preserve deterministic, reproducible behavior

**The entropy tax:**
Per 100 lines added → identify 10 removable → identify 1 simplification → identify 1 future extraction boundary.

**The objective is not feature velocity.**
The objective is **equilibrium** — a system that remains resilient, observable, maintainable, and recoverable indefinitely.

**Prefer constraints over logic. Prefer removal over expansion.**

## 6. ARCHITECTURE RULES

### 6.1 Layer Discipline

Dependencies flow **downward only**.

- No upward imports
- No cross-layer mutation
- No hidden coupling
- No shared mutable state

### 6.2 Eventual Consistency

Systems must survive subsystem outages.

- Events over direct calls
- Queues over synchronous chains
- Replayable state transitions
- No cascading failure paths

### 6.3 Deterministic State

Every transition reproducible from inputs + events + logs.

- No invisible mutation
- No hidden global state
- No timing-dependent behavior

### 6.4 Flat Complexity

- Shallow modules
- Explicit composition
- Local reasoning
- No deep inheritance
- No magical abstractions
- No framework traps

### 6.5 Five-Second Readability Rule

A function must be understandable in one glance, one screen, five seconds.
If not → split it. No exceptions.

### 6.6 Module Size Contract

- 40 lines of dense logic → extraction review mandatory
- 200 lines per file → architectural review mandatory
- One file = one responsibility. Always.

### 6.7 Dependency Hygiene

- No circular dependencies. Ever.
- No orphan utilities.
- No implicit peer dependencies.
- Every external dependency must justify its presence.

## 12. SECURITY MODEL

**Default posture: Zero Trust. Least Privilege. Explicit Validation.**

Requirements:

- Input validation on every external boundary (Zod schema enforcement)
- Structured auth boundaries — no implicit trust propagation
- Secrets isolation — env vars only, never in source, never in logs
- Rate limiting on all public surfaces
- Replay protection on all mutation operations
- Structured audit logs for all auth events
- HMAC signing for all cross-service events
- AES-256-GCM for all sensitive payloads at rest

Never trust:

- Network stability
- User-supplied input without validation
- External service availability
- Timing assumptions
- Token freshness without explicit verification

## 13. PERFORMANCE MODEL

**Primary metrics:** latency · throughput · memory pressure · cold start cost · query efficiency · render stability

**Optimization order (never skip steps):**

1. Eliminate unnecessary work entirely
1. Reduce memory allocations
1. Reduce synchronization surface
1. Reduce network dependency
1. Parallelize safely with backpressure

**Database:**

- Every PostGIS query requires GIST index verification
- Prisma connection pool sized to VM: `(2 × CPUs) + 1`
- Raw queries only for spatial operations
- No N+1 patterns — ever

**React Native:**

- No blocking operations on JS thread
- Reanimated worklets for all animations
- `useMemo`/`useCallback` on all expensive computations
- FlatList with `getItemLayout` for all long lists

## 14. OBSERVABILITY MODEL

Every system must expose:

- Structured JSON logs (machine-readable first)
- Distributed trace IDs on all cross-service calls
- Prometheus-compatible metrics
- Event history with replay capability
- Failure states with structured error objects
- Recovery attempt logs with outcomes

**Log structure:**

```typescript
{
  timestamp: ISO8601,
  traceId: string,
  service: string,
  level: 'debug' | 'info' | 'warn' | 'error' | 'fatal',
  event: string,
  data: Record<string, unknown>,
  duration_ms?: number,
  error?: { code: string; message: string; stack?: string }
}
```
